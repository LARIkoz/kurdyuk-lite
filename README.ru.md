# Kurdyuk Lite

**Фреймворк для продуктовой разработки с AI-агентами, управляемый Charter (уставом).**

Хватит _болтать_ с AI-агентом. Начните _направлять_ его — через строгий шестифазный пайплайн с письменной конституцией в основе. Нативно для Claude Code; работает с Codex, Gemini и любым агентом, который читает файл инструкций проекта.

```
Brainstorm+Research+Brief → Charter → Spec → Plan → Implement → Final Review
```

## Проблема

Вы открываете Claude Code с реальной продуктовой идеей. Три часа спустя у вас гора полусобранных фич, цель, которая тихо сдвинулась, и агент, который «улучшил» то, о чём вы не просили. Контекст потерян. Решение, принятое в два часа дня, не зафиксировано нигде — ни в файле, ни в памяти агента после компактизации. Вы переделываете работу, которую AI уже сделал, потому что он ушёл в сторону.

Свободный чат с мощным агентом — быстрый и затягивающий — и он обходится дорого: **потерянный контекст, расползание скоупа, тихий дрейф и ни одной записи о том, _почему_ что-то было решено.**

## Что делает Kurdyuk Lite

Он ставит **рельсы** под работу. Шесть фаз, каждая порождает один артефакт, каждая проходит только с вашим явным «ОК». Нельзя пропустить. Нельзя объединить. Никаких «я просто начну кодить».

В центре — **Charter** (устав) — короткая нумерованная конституция продукта. Не спека, не бэклог: _принципы_. Порядок = приоритет. Каждое проектное и реализационное решение проверяется по нему; нарушение — это **блокер**, а не дискуссия.

Агент возобновляет работу из **файлов, а не из памяти чата** — поэтому `/compact`, новая сессия или недельный перерыв не обрывают нить.

## Какую задачу это закрывает

Мы проектировали Kurdyuk Lite вокруг одной **Большой задачи** (в терминах Jobs-to-be-Done — той высшей цели, ради которой вы нанимаете инструмент):

> **«Выпустить продукт, за который не стыдно — построенный в основном с AI-агентом — не теряя над ним контроль и не выгорая.»**

У этой задачи три слоя, и фреймворк обслуживает все три:

- **Функциональный** — превратить идею в работающее, связное ПО при помощи агента.
- **Эмоциональный** — ощущать _контроль_, а не тревогу, что агент тихо ушёл куда-то не туда.
- **Социальный** — создавать работу, которую можно показать коллеге или пользователю и **обосновать**: «вот _почему_ это сделано так».

Большая задача никогда не закрывается одним инструментом полностью. Под ней — **Ключевая задача**, которую Kurdyuk Lite действительно закрывает:

> **Провести дисциплинированный процесс от идеи до ревью с AI-агентом, чтобы ничего важного не было импровизировано или потеряно.**

А **критическая последовательность** — шаги, без которых всё рассыпается — _и есть_ шесть фаз:

| Фаза             | Какую задачу закрывает                                                  |
| ---------------- | ----------------------------------------------------------------------- |
| **Brief**        | Договориться, что строим, для кого и что вне скоупа                     |
| **Charter**      | Зафиксировать принципы, по которым судят все последующие решения        |
| **Spec**         | Превратить принципы в конкретные, проверяемые требования                |
| **Plan**         | Разбить спеку на упорядоченные, верифицируемые задачи                   |
| **Implement**    | Реализовать, доказав выполнение каждой задачи — без имитации завершения |
| **Final Review** | Проверить по Charter + Spec; вынести вердикт `go / revise / no-go`      |

Убери Charter — и агент оптимизирует в вакууме. Убери гейты — и скоуп расползётся. Каждый файл зарабатывает своё место.

### Почему вы переключитесь (четыре силы)

Переход от _«просто чатить с агентом»_ к _«направлять его через пайплайн»_ — это поведенческое переключение, и переключения подчиняются четырём силам:

- **Выталкивание (Push)** — что болит сегодня: контекст испаряется, агент уходит от задачи, скоуп расползается, нет записей о том, _почему_. Каждый пункт — это лишняя работа, которую вы не планировали.
- **Притяжение (Pull)** — что тянет к новому: контекст, который переживает фазы и компактизации, решения, закреплённые в Charter, на который можно сослаться, дрейф, пойманный в момент нарушения принципа.
- **Привычка (Habit)** — что держит на чате: рефлекс сразу начать кодить, мышечная память «просто спрошу у агента», вера в то, что всё запомните.
- **Тревога (Anxiety)** — что заставляет колебаться: _не замедлит ли это меня? не превратится ли Charter в бюрократию? сможет ли агент вообще следовать процессу?_

Kurdyuk Lite намеренно **Lite**, чтобы побить последние две силы: один контекст, восемь Markdown-шаблонов, гейты, которые проходятся за секунды. Дисциплина настоящая; оверхед — нет.

> **Честная оговорка.** Это _дизайн-гипотеза_, под которую мы строили фреймворк, сформулированная на языке Jobs-to-be-Done (школа AJTBD) — **не** валидированное пользовательское исследование. Если вы попробуете — ваша реальность и будет настоящей проверкой.

## Как выглядит прогон

1. **Brief** — вы проводите мозговой штурм с агентом; он сводит результат в `brief.md` (определение в одну строку, для кого, что _не_ в скоупе) и ждёт вашего «brief OK».
2. **Charter** — агент синтезирует нумерованные принципы в `charter.md`. Вы ревьюите душу продукта прежде, чем появится хоть строчка спеки.
3. **Spec** — принципы превращаются в функциональные/нефункциональные требования с критериями приёмки, каждое прослеживается до принципа.
4. **Plan** — спека превращается в упорядоченные задачи с условиями «сделано, когда» в `plan.md` / `todo.md`.
5. **Implement** — агент работает задача за задачей, доказывая выполнение каждой. Если решение нарушает Charter, агент останавливается и сигнализирует.
6. **Final Review** — вердикт по каждому принципу и каждому критерию приёмки.

Каждая фаза обновляет `state.md` (источник истины проекта) и дописывает в `gate-log.md` (след одобрений). Возобновить работу можно в любой момент, прочитав эти два файла.

## Charter подробнее

Charter — это конституция, написанная по четырём правилам (см. [`charter-for-charter.md`](charter-for-charter.md)): **никаких эвристик, никакого КАК, каждый принцип — это тест, напиши и сокращай.** Нумерованные, упорядоченные по приоритету, не более 3 предложений каждый. Пример из проработанного [`reference/example-charter.md`](reference/example-charter.md):

```
1. The app exists to make money. Persuasion is business; deception is debt.
2. One tap to focus. Starting a session must never require setup.
3. The timer is sacred. A running session is never interrupted by upsells.
4. Honest time. Reported focus time reflects real elapsed focus only.
5. Works offline. Core timing functions with no network.

When principles conflict, lower number wins.
```

Теперь на вопрос «стоит ли поставить пейволл на стартовый экран?» есть ответ — _нарушает P2_ — вместо совещания.

## Установка

### В одну строку — пусть агент установит сам

Вставьте это в любой AI-агент для кода (Claude Code, Codex, Gemini CLI, ...), открытый в вашем проекте:

> **Install Kurdyuk Lite in this repo: follow `https://raw.githubusercontent.com/LARIkoz/kurdyuk-lite/main/BOOTSTRAP.md` — pick the `--agent` that matches you, then run `check` and report the Status line.**

Агент клонирует фреймворк, инъецирует контракт в _свой_ файл инструкций (`CLAUDE.md` / `AGENTS.md` / `GEMINI.md`), проверяет установку и выдаёт вердикт — ноль ручных шагов, любой агент. Инструкция, которой он следует: [`BOOTSTRAP.md`](BOOTSTRAP.md).

### Вручную

```bash
git clone https://github.com/LARIkoz/kurdyuk-lite.git ~/.kurdyuk-lite
~/.kurdyuk-lite/bin/kurdyuk-lite init .      # injects the contract into ./CLAUDE.md + copies templates
~/.kurdyuk-lite/bin/kurdyuk-lite check .     # → Status: UP TO DATE
```

Или клонировать + init + проверить одной командой, выбрав файл агента:

```bash
curl -fsSL https://raw.githubusercontent.com/LARIkoz/kurdyuk-lite/main/bin/kurdyuk-install | bash -s -- --agent claude .
```

`init` идемпотентен и неразрушителен: он инъецирует маркированный блок в файл инструкций вашего проекта (предварительно создав бэкап), кладёт шаблоны в `.kurdyuk-lite/` и обновляет `.gitignore`. Повторный запуск обновляет фреймворк.

## Работает с любым агентом

Контракт — это обычный Markdown. Любой агент для кода, который загружает файл инструкций проекта, может его выполнить. `init` записывает файл для выбранного агента:

```bash
kurdyuk-lite init .                 # Claude Code  → CLAUDE.md   (default)
kurdyuk-lite init . --agent codex   # Codex / others → AGENTS.md
kurdyuk-lite init . --agent gemini  # Gemini CLI   → GEMINI.md
kurdyuk-lite init . --agent all     # all three at once
```

`AGENTS.md` — кроссинструментальный стандарт, поэтому `--agent codex` покрывает и другие агенты, поддерживающие AGENTS.md. Используете что-то другое (Cursor, Aider, ...)? Укажите агенту на `CLAUDE-contract.md` или скопируйте инъецированный блок в файл инструкций вашего инструмента. `check` работает одинаково вне зависимости от того, в каком файле находится контракт.

## Быстрый старт

Откройте Claude в вашем репозитории и либо вставьте `.kurdyuk-lite/STARTER-PROMPT.md`, либо просто опишите, что хотите построить. Агент подтвердит режим Kurdyuk Lite, создаст папку прогона и начнёт Phase 1.

## Когда использовать

- **3-7 задач, 1-2 сессии** — Kurdyuk Lite в своей стихии.
- **8+ задач, мультисессионный проект, гейты валидации дизайна** — вы переросли Lite; разбейте работу или используйте более тяжёлый процесс.
- **Однострочный фикс или быстрый рефакторинг** — фреймворк не нужен. Не стоит нести конституцию к опечатке.

## Что в комплекте

```
kurdyuk-lite/
├── CLAUDE.md              # thin wrapper → points at the contract
├── CLAUDE-contract.md     # the canonical operating contract (injected into your project)

├── reference/             # worked charter examples + optional discovery toolkit (not copied into your project)
└── templates/             # the 8 phase templates init copies into .kurdyuk-lite/templates/
```

## FAQ

**Что такое Kurdyuk Lite?**
Фреймворк для продуктовой разработки с AI-агентами, управляемый Charter. Он проводит вашего AI-агента через шесть гейтованных фаз — Brief, Charter, Spec, Plan, Implement, Final Review — с нумерованным Charter (конституцией продукта) в центре.

**Чем это отличается от обычного использования Claude Code?**
Обычный чат открыт: агент может перескочить вперёд, уйти в сторону и забыть. Kurdyuk Lite добавляет гейты (каждая фаза требует вашего «ОК»), Charter (каждое решение проверяется по письменным принципам) и файловое состояние (работа переживает `/compact` или новую сессию).

**Работает ли с Cursor, Codex, Gemini или другими AI-агентами?**
Да. Контракт — это обычный Markdown. `init --agent codex` записывает `AGENTS.md`, `--agent gemini` — `GEMINI.md`, `--agent all` — все файлы сразу. Для всего остального укажите агенту на `CLAUDE-contract.md`. Claude Code — агент по умолчанию и наиболее протестированный.

**Процесс меня не замедлит?**
Гейты проходятся за секунды; Charter — это несколько строк. Время, которое он экономит — не переделывать работу, от которой агент ушёл в сторону — и есть суть. Для однострочного фикса можно обойтись без него.

**Можно добавить в существующий проект?**
Да. `init` неразрушителен: он инъецирует маркированный блок в существующий `CLAUDE.md` (предварительно создав бэкап) и добавляет папку `.kurdyuk-lite/`. Повторный запуск обновляет фреймворк.

**Мой код или данные куда-то отправляются?**
Нет. Kurdyuk Lite — это Bash + Markdown без сетевых вызовов и телеметрии. Он только пишет файлы в вашем репозитории; всё остальное происходит внутри вашей собственной сессии Claude Code.

**Что такое Charter в одной строке?**
Короткая нумерованная конституция вашего продукта — порядок = приоритет — которая превращает «стоит ли делать Х?» в проверяемый ответ («нарушает P2») вместо дискуссии.
