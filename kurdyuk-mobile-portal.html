Phase Details

Раскрой каждую фазу для подробностей

0

Discovery + Market Research

▼
  • 📋 Brief: 4 опоры (что / для кого / что НЕ / первый успех)
  • 🔍 ASO keyword research (Gemini bg) — влияет на naming
  • 📱 Competitor screenshots (app-store-scraper) → Claude анализирует
  • 💰 Monetization model (подписка / разовая / freemium + что за paywall)
  • 📤 Output: brief.md, research.md, reference-screenshots/
1

Xcode Setup + Foundation

▼
  • ⚙ Xcode project creation (File → New Project)
  • 🔑 Code Signing & Provisioning Profiles
  • 🛡 Privacy Manifests (NSPrivacyAccessedAPITypes)
  • 📝 Info.plist: usage descriptions
  • 📦 SPM: RevenueCat, Firebase/TelemetryDeck
  • ⚠ Health apps = 1-2 дня setup, не "10 минут"
2

IA + Data Model

▼
  • 🗺 Screen list + navigation flow
  • 💾 Data model sketch (entities, relationships)
  • 🛒 Paywall placement (после какой фичи?)
  • 🌐 [IF 10+ screens] HTML wireframe portal
  • 📄 Output: screen-map.md, data-sketch.md
3

Design Exploration (FORK)

▼
  • 🔴 Развилка: Mode A (Quick) / Mode B (Boosted) / Mode C (Polish Loop)
  • 🎨 3 visual styles одной структуры с named references
  • 📱 2-3 key screens per style в SwiftUI с #Preview
  • 👀 Review в Xcode Canvas → user picks
  • 🛑 ЛИМИТ: 3 стиля, 1 feedback round
4

Implementation

▼
  • 🏗 Core Architecture (Observation + NavigationStack + SwiftData)
  • 🎨 Screens 1-2: hardcode. Screen 3: EXTRACT DesignSystem
  • ⚙ Core features + Widgets + StoreKit 2 / RevenueCat
  • 📊 Analytics events (screen_view, button_tap, purchase_made)
  • 🛡 System APIs (HealthKit, iCloud, Notifications)
  • ∅ All states: empty, error, offline
  • 📱 Per-screen: design-checklist.md
  • 📷 [IF Standard+] snapshot regression каждые 3 экрана
5

Polish (30-50% времени!)

▼
  • ✨ Animations (.spring, PhaseAnimator, MeshGradient)
  • 📳 Haptics (.sensoryFeedback) — AI бесполезен, руками
  • 🌙 Dark mode audit
  • ∅ Empty states с иллюстрациями
  • 🎨 App Icon (DALL-E API)
  • ♿ Accessibility (Dynamic Type, VoiceOver, contrast)
  • ⚠ WARNING: AI = черновик, человек = доводка
6

ASO + Submit

▼
  • 📝 App Store metadata (title, subtitle, keywords, description)
  • 📷 Screenshot generation (simctl + device frames)
  • ⚙ App Store Connect setup
  • 🛡 Privacy nutrition labels
  • 🚀 fastlane deliver → TestFlight → App Store
7

Iteration & Analytics

▼
  • 📊 Аналитика → бэклог: "80% отвал на экране X → редизайн"
  • ⭐ Отзывы App Store + TestFlight → задачи v1.1
  • 🧪 A/B тесты (два paywall'а → данные → решение)
  • 💰 Revenue metrics → корректировка pricing/features
  • 🔁 Цикл: Phase 7 → Phase 0 (новый brief для v1.1)

Design Modes

Phase 3 развилка: скорость vs качество. Claude спрашивает в Phase 0.

MODE A

Quick

~30 мин • ~7-8/10 • $0
  • 1app-store-scraper → скриншоты конкурентов
  • 2Claude Opus → vibe extraction
  • 33 visual styles + Cookbook recipes
  • 42-3 screens per style в SwiftUI #Preview
  • 5Xcode Canvas → pick
Для: MVP, hypothesis testing, speed
MODE B

Boosted

~2-3 ч • ~9/10 • ~$0-5
  • 1Pinterest API → design references
  • 2app-store-scraper → competitor screenshots
  • 3Claude Vision → extract palette, patterns
  • 4Recraft v3 API → 3-5 concept mockups
  • 5Claude sees Recraft PNG → SwiftUI matching
  • 6Xcode Canvas → pick
Для: Store product, premium feel, monetized app
MODE C

AI Polish Loop

After A or B • max 5 cycles • $0
  • 1Claude Sonnet → SwiftUI build
  • 2xcodebuild + simctl screenshot
  • 3Gemini Pro (bg) critiques screenshot
  • 4Claude reads critique → applies fixes
  • 5Repeat until Gemini says "9/10, stop"
Key: Claude = executor, Gemini = independent critic

Architect

Key Principles

7 правил, которые определяют весь pipeline.

1

Design = Code

SwiftUI-first, не Figma. #Preview = главный инструмент дизайна.

2

DS = EXTRACT

Из 3-го экрана, не из воздуха. Органически, не формально.

3

Prompt > APIs

Правильный промпт с Cookbook = 9/10. Recraft/Pinterest = backup.

4

Monetization from P0

StoreKit/RevenueCat с нулевой фазы. Без денег = хобби.

5

Widgets = Core

Для trackers/utility — главный канал retention после дня 1.

6

Polish = 30-50%

AI = черновик, человек = магия. Анимации, haptics, dark mode.

7

Phase 7 → Phase 0

Risks & Mitigations

Risk Impact Mitigation
No monetization = hobby project Critical Phase 0: модель. Phase 4: StoreKit/RevenueCat
"Safe" average designs from AI High Category-aware references (Calm, not Things 3 for mood)
Polish = 30-50%, not 10% High Budget awareness. AI = черновик, человек = магия
No analytics = blind High Phase 1: SDK. Phase 4: events
Widgets treated as "optional" High Phase 4: CORE for trackers. App Groups в Phase 1
Privacy Manifests forgotten High Checklist в Phase 1 + Phase 6
Design phase scope creep Medium Hard limit: 3 styles, 1 feedback round
AI "hallucinations" in SwiftUI Medium Every output → run in Preview, not blind copy
Code Signing hell Medium Phase 1: do it IMMEDIATELY. Health apps = 1-2 days
