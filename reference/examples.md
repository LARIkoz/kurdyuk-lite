# Reference Charters

Charters to study before writing your own. Start with the worked example, then the meta-charter.

### `reference/example-charter.md` — complete worked example

A full, self-contained charter (a generic focus-timer product) showing every section: Preamble → Principles →
Scope → Good/Bad/FP examples → Architecture constraints → Success criterion. Read this first.

### `charter-for-charter.md` — the meta-charter

The 4 rules that govern every charter: no heuristics, no HOW, every principle is a test, write-then-cut.

### `reference/charter-product-discovery-example.md` — research-pipeline variant

A charter template for a product built on top of a research pipeline (market scan → user research → job map →
competitor analysis). Shows how evidence informs the founder's principles without entering the charter. Not
copied into target projects by `init` — use it only when your project sits behind a research pipeline.
